Projects I've Enjoyed

I have most recently been working with a SaaS company creating an advertisement bidding platform that allows hotels to place bids with Google and TripAdvisor through an easy to use admin. Through this position I have been able to work with several APIâ€™s including Google Travel Partners and TripAdvisor.

Analytics

One of the more recent projects I worked on was a scheduled job that downloaded the Google Ad Reports for each of the hotels using our service and imported the data into our database. The data was then run through several algorithms to produce more usable bid analytics view able through a clean UI.

API Endpoints

Another recent project I worked on was creating several API endpoints to be used by a Angular.js front end application. I found this work to be greatly enjoyable and was a great opportunity to practice TDD.

Redis

One of the more complicated projects I built was combining the use of a Redis storage and a Resque job. The job would process hotels that needed to be notified of exceeding monthly budgets and if the budget was exceeded the ids would be stored in Redis and saved until other notification jobs have ran. Then the Redis set would be processed by a task to build up a notifications email list.

A Learning Project

I have been working on an application that allows a user to register clients via a client email that gets pushed into a queue to be processed by a task that gathers data based on the clients email address. An initial registration POST is sent that queues the client for a task that gets ran every day. A GET request can then be made to retrieve the recently produced client information. This is still a work in progress but the login and API are up and running. This has mostly been a side project to get more familiar with authentication for APIâ€™s as well as eventually working on an Angular.js app that will access this API.

Metrics and Bugs

I like to write features that help people understand things more clearly. This could mean designing an easier to use UI or creating business metrics that allow users to see data in a new light and in turn help them take controller of another aspect of their business.
I also really love tracking down bugs. I like chasing through the code and understanding what makes a feature work, or in most cases why an Airbrake error is being sent. I also have found a new interest in testing. I really like Rspec and FactoryGirl and have found that writing a quick test can actually help to restrict variables in order to track down bugs more easily.
